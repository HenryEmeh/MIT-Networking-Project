Batch 35 – Agricultural IoT & Smart-Farming Connectivity
Scenario:
AgroLink Nigeria deploys IoT sensors for soil moisture, drones for crop imaging,
and cloud dashboards for analytics.
Rural farms face unstable power, weak connectivity, and limited security controls.
You must design a resilient, secure, and bandwidth-optimised smart-farming
network aligned with NDPR and IoT best practice.
1. Farm Network Topology (Diagram)
Instruction: Sketch field sensors, gateways, cloud server, and admin dashboard
links; show redundancy.
Evaluation Focus: L1 topology realism.
2. IP/VLAN Schema (VLSM)
Instruction: Allocate subnets for Sensors, Drones, Admin, and Guest Wi-Fi;
summarise addressing.
Evaluation Focus: L2 subneing mastery.
3. Routing between Farm & HQ
Instruction: Use dynamic routing over L TE/VSAT with route failover; include
example metric seings.
Evaluation Focus: L2 path-control planning.

4. Firewall & Access-Control List
Instruction: Restrict sensor communication to authorised MQTT brokers.
Evaluation Focus: L3 ACL precision.
5. QoS for Telemetry Traic
Instruction: Prioritise sensor telemetry over video streams; define DSCP
mapping.
Evaluation Focus: L4 latency-management skill.
6. SDN Automation
Instruction: Automate addition/removal of field gateways using controller APIs.
Evaluation Focus: L5 automation agility.
7. Edge Computing Node
Instruction: Deploy edge devices performing data filtering and anomaly detection
locally.
Evaluation Focus: L6 edge-analytics deployment.
8. Monitoring & Dashboard
Instruction: KPIs—sensor uptime, link latency, energy use—and threshold alerts.
Evaluation Focus: L4 observability design.
9. DR & Backup Connectivity
Instruction: Provide backup link using secondary ISP or satellite.
Evaluation Focus: L1 continuity.
10. Power Resilience Plan
Instruction: Integrate solar panels and smart baeries; estimate runtime.
Evaluation Focus: L1 sustainability planning.
11. NDPR Compliance
Instruction: Ensure farmer-data anonymisation and secure API access.
Evaluation Focus: L3 privacy control.
12. Incident-Response Guide
Instruction: Outline procedure for compromised IoT gateway isolation.
Evaluation Focus: L3/L4 IR execution.
13. Cost-Benefit Analysis
Instruction: Calculate savings from edge data reduction vs cloud cost.
Evaluation Focus: L4 analytical thinking.
14. Training & User Support
Instruction: Develop farmer-friendly manual on system maintenance.
Evaluation Focus: L3 education design.
15. Scalability & Expansion Plan
Instruction: Plan expansion from 3 to 10 farms with address and bandwidth
forecasting.
Evaluation Focus: L1/L4 growth modelling.
